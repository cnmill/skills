# 产业链/技术链深度调研

## 触发条件

- 用户提到一家非上市公司（如曦智科技、Lightmatter等），要求分析其经营范围和A股对标
- 用户提到一个技术概念（如光互连、CPO、DSP芯片、旋片等），要求梳理产业链
- 用户要求"产业拓扑图"、"产业链全景"、"A股对标企业"
- 用户要求分析某技术的上下游和投资映射

## 工作流程

### 第一步：并行信息收集（delegate_task，3路并发）

```
Task 1: 目标公司/技术深度调研
  - 公司背景、创始团队、融资历程
  - 核心技术路线和原理
  - 产品线和商业化进展
  - 最新动态和战略方向

Task 2: 产业链全景调研
  - 技术路线对比（如光互连vs电互连、CPO vs可插拔）
  - 上下游产业链（材料→芯片→模块→系统）
  - 全球主要玩家（海外+国内）
  - 市场规模和增长预期（TAM必须估算）

Task 3: A股对标企业搜索
  - 按产业链环节列出相关上市公司
  - 每家公司：代码、主营中相关部分、市值、近期表现
  - 重点关注用户指定的标的
```

### 第二步：整合分析与报告生成

报告结构：
1. **目标公司/技术概览**（表格：维度/内容）
2. **核心技术路线**（原理、演进、对比表格）
3. **产业链拓扑图**（ASCII art，上游→中游→下游+新兴分支）
4. **市场规模**（表格：细分/当前/预期/CAGR）
5. **A股对标企业**（按产业链位置分梯队）
6. **投资视角分层**（主线核心/弹性标的/国产替代/低位预期差）
7. **核心结论**（3-4条）

### 第三步：持久化

```bash
# 保存到 hermesfile 对应产业目录
/Users/mac/Documents/own/hermesfile/research/{产业名}/{报告标题}.md

# git push
cd /Users/mac/Documents/own/hermesfile && git add -A && git commit -m "add {报告标题}" && git push
```

## 产业拓扑图规范

使用 ASCII art 绘制，结构：
```
上游（核心材料与芯片）
  ├─ 材料层
  ├─ 芯片/器件层
      ↓
中游（模块与引擎）
  ├─ 产品形态A
  ├─ 产品形态B
      ↓
下游（系统与应用）
  ├─ 应用场景列表

新兴分支（如有）
  ├─ 前沿技术方向
```

## A股对标分层方法

| 类型 | 定义 | 特征 |
|------|------|------|
| 主线核心 | 产业链最确定性环节的龙头 | 市值大、业绩兑现、机构重仓 |
| 弹性标的 | 中期爆发方向的核心受益者 | 技术卡位好、商业化临近 |
| 国产替代 | 卡脖子环节的突破者 | 国产化率低、替代空间大 |
| 低位预期差 | 逻辑成立但市场未充分定价 | 小市值、高弹性、需催化 |

## 技术对比表格规范

对比维度应包含：带宽/功耗/成本/成熟度/适用场景/趋势判断。
每个维度给出定量数据（如"功耗降低30-50%"），避免纯定性描述。

## 注意事项

- TAM/市场规模必须估算，用户明确要求"较大终端市场才能支撑高增长"
- 过滤掉纯指数类/重仓类/区域类概念，只保留真正的题材
- 区分"已被充分交易的强趋势"vs"低位预期差"vs"纯事件脉冲"
- 非上市公司分析时注明其与A股标的的竞争/合作关系
- 国产替代进展要诚实标注差距（如"落后2-3代"），不要过度乐观
- BOM价值占比是判断产业链话语权的关键指标，尽量给出
