# 问财行情数据查询

## 前置条件

1. 检查是否已安装 Iwencai SkillHub CLI：
```bash
command -v iwencai-skillhub-cli || command -v iwencai || echo "未安装"
```

2. 若未安装，执行安装脚本（仅安装 CLI）：
```bash
curl -fsSL https://www.iwencai.com/skillhub/static/0.0.4/download_and_install.sh | bash
```

3. 安装 `行情数据查询` 技能：
```bash
iwencai-skillhub-cli install 行情数据查询 || iwencai install 行情数据查询
```

4. 确保环境变量已配置：
```
IWENCAI_BASE_URL=https://openapi.iwencai.com
IWENCAI_API_KEY=<用户的API Key>
```

## 使用场景

- 查询个股/板块实时及历史行情
- 筛选涨幅/跌幅/成交量异动股票
- 技术面数据获取（均线、MACD等）
- 概念板块涨幅排名、行业对比

### 常用查询语句示例

- `'概念板块近1年涨幅排名从高到低'` — 概念板块涨幅排名
- `'同花顺最新价格'` — 个股价格
- `'主力资金流向'` — 资金流向
- `'上证指数行情'` — 指数行情
- `'PCB概念板块成分股涨幅排名'` — 特定板块成分股

### 返回数据结构

JSON 格式，关键字段：
- `datas`: 数据数组，每条记录为一个对象
- `code_count`: 总匹配数（可能大于当前页返回量）
- 板块查询常见字段：`指数简称`、`涨跌幅[日期范围]`、`指数代码`

### 注意事项

- 默认 limit=10，查板块排名建议设 limit=100
- code_count 显示总数，需翻页合并完整数据
- 返回字段名可能包含日期范围如 `涨跌幅[20250428-20260427]`，需动态解析
- 如果子任务调用，需在命令中内联环境变量：`IWENCAI_API_KEY='...' python3 ...`

## 宏观/主题研究工作流

当用户不是要查单只股票，而是要把一个宏观主题、气候事件、政策方向或产业催化映射到A股时，优先按下面步骤做：

1. **先拆冲击链，不要直接搜主题名**
   - 先把主题拆成一阶、二阶、三阶影响。
   - 例如：高温/厄尔尼诺 -> 用电负荷 -> 电网扩容/变压器/配电；
     高温 -> 制冷需求 -> 热管理/压缩机/换热器；
     干旱/洪涝 -> 水利/灌溉/排涝；
     农业减产预期 -> 种业/农药。

2. **用多个代理概念词验证“市场温度”**
   - 不要只查一个概念板块，优先并行查多个最相关代理方向的近1年涨幅或成分股表现。
   - 常用问句模板：
     - `'XX概念板块近1年涨幅排名从高到低'`
     - `'XX概念板块成分股近1年涨幅从高到低'`
     - `'主营业务包含XX的A股，近1年涨幅从高到低'`
   - 用来判断该方向是：
     - 已被充分交易的强趋势
     - 低位预期差
     - 纯事件脉冲

3. **概念查询和主营查询要结合使用**
   - 概念板块适合看题材热度、市场风格、成分股排序。
   - 主营业务查询适合提高“业务纯度”，避免概念股过度泛化。
   - 若某主题概念指数过窄、样本过少或失真，改用主营口径补充验证。

4. **优先做横向比较，而不是孤立解读单个板块**
   - 至少比较 3 个相关方向，例如：
     - 电力链：`变压器` / `智能电网` / `特高压`
     - 制冷链：`空气能热泵` / `液冷服务器` / `空调`
     - 农业链：`种业` / `农药` / `节水灌溉`
     - 水利链：`水利建设` / `地下管网` / `排涝` / `智慧水利`
   - 目标不是找“唯一正确概念”，而是看哪条分支最强、哪条最滞涨、哪条最有扩散空间。

5. **输出结论时区分三种类型**
   - `主线核心`：景气度高、市场规模大、具备持续性
   - `高弹性补涨`：主线扩散但位置相对靠后
   - `低位预期差`：逻辑成立但市场尚未充分定价

6. **把市场规模/TAM单独写出来**
   - 用户若在做主题投资，不要只给“受益逻辑”和“涨幅排序”。
   - 需补一层：这个方向的市场空间大不大，能否支撑高增长。
   - 实操上可按：小赛道 / 数百亿级 / 千亿级 / 万亿级 投资链条进行定性分层。

## API 不可用时的降级策略

问财 API 可能因权限过期、限流或服务端问题返回 `无请求权限` 错误。遇到此情况不要反复重试不同查询口径（通常全部会失败），而是立即降级：

1. **优先从 hermesfile 已有研究文件中提取标的池**
   - 搜索 `/Users/mac/Documents/own/hermesfile/research/` 下已有报告中的个股名称和代码
   - 已有报告通常包含经过验证的标的映射（如 AI链条、电力链条、算力产业等）
   - 用 `search_files` 按公司名批量搜索，快速汇总已有标的覆盖

2. **利用已缓存的概念板块涨幅数据**
   - `hermesfile/research/概念板块1年涨幅排名.md` 包含388个概念板块的近1年涨幅快照
   - 可直接用于判断板块市场温度，无需实时API

3. **用 web_search 补充关键数据点**
   - 对于必须获取的实时数据（如最新财报、重大事件），可用 web_search 做定向补充

4. **标记数据时效性**
   - 降级产出的报告中应注明数据来源和时效（如"基于已有研究文件交叉验证，非实时行情"）
